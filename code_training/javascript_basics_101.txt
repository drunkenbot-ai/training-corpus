# Variables: let, const, var

## Overview

JavaScript uses three keywords to declare variables: `var`, `let`, and
`const`. Modern JavaScript code almost always prefers `let` and `const`
over the older `var`.

## Key Concepts

`var` is function-scoped and can be redeclared, which historically caused
bugs. `let` is block-scoped and can be reassigned. `const` is also
block-scoped but cannot be reassigned after its initial value is set,
though the contents of an object or array stored in a `const` can still be
changed.

## Syntax

```javascript
var oldStyle = 1;
let counter = 0;
const maxValue = 100;
```

## Examples

### Example 1

```javascript
let score = 10;
score = score + 5;
console.log(score);
```

`let` allows reassignment, so `score` becomes 15.

### Example 2

```javascript
const person = { name: "Ada" };
person.name = "Grace";
console.log(person);
```

`const` prevents reassigning `person` itself to a new object, but the
properties inside the object can still be modified.

### Example 3

```javascript
if (true) {
  let blockScoped = "inside";
  var functionScoped = "also inside";
}
console.log(functionScoped);
```

`var` leaks out of the `if` block because it is function-scoped, while a
`let` declared the same way would not be accessible outside the block.

## Common Mistakes

```javascript
const total = 10;
total = 20;
```

This raises `TypeError: Assignment to constant variable`, because `const`
does not allow reassignment. Use `let` instead if the variable's value
needs to change later.

## Best Practices

Default to `const` for any variable that will not be reassigned, and use
`let` only when reassignment is genuinely needed. Avoid `var` in new code,
since its function-scoping rules are a common source of bugs.

# Data Types

## Overview

JavaScript has a small set of primitive data types, plus objects, which
cover everything from arrays to functions.

## Key Concepts

The primitive types are `number`, `string`, `boolean`, `undefined`,
`null`, `bigint`, and `symbol`. JavaScript uses a single `number` type for
both integers and decimals, backed by double-precision floating point.
Everything that is not a primitive is an object, including arrays and
functions.

## Syntax

```javascript
let age = 30;
let name = "Ada";
let isActive = true;
let nothing = null;
let notSet;
```

## Examples

### Example 1

```javascript
console.log(typeof 42);
console.log(typeof "hello");
console.log(typeof true);
console.log(typeof undefined);
console.log(typeof null);
```

The `typeof` operator reports each value's type as a string; notably,
`typeof null` returns `"object"`, which is a well-known historical quirk
of the language.

### Example 2

```javascript
let price = 19.99;
let quantity = 3;
console.log(price * quantity);
```

Since there is only one `number` type, integers and decimals can be mixed
freely in arithmetic without explicit conversion.

### Example 3

```javascript
let bigNumber = 9007199254740993n;
console.log(typeof bigNumber);
```

`BigInt`, created by appending `n` to a numeric literal, represents
integers larger than `Number` can represent exactly.

## Common Mistakes

```javascript
let total = "5" + 3;
console.log(total);
```

This prints the string `"53"` instead of the number `8`, because `+`
performs string concatenation when either operand is a string. Use
`Number("5") + 3` if numeric addition is intended.

## Best Practices

Use strict equality (`===`) rather than loose equality (`==`) to avoid
unexpected type coercion during comparisons. Convert user input explicitly
with `Number()` or `parseInt()` before doing arithmetic on it.

## Related Topics

Variables, Operators, Strings

# Strings

## Overview

Strings represent text in JavaScript and come with many built-in methods
for searching, transforming, and combining text.

## Key Concepts

Strings are immutable; every method that appears to modify a string
actually returns a new string. Template literals, written with backticks,
allow embedded expressions and multi-line text.

## Syntax

```javascript
let greeting = "Hello";
let name = 'World';
let message = `${greeting}, ${name}!`;
```

## Examples

### Example 1

```javascript
let text = "  JavaScript is fun  ";
console.log(text.trim());
console.log(text.trim().toUpperCase());
```

`trim()` removes surrounding whitespace, and `toUpperCase()` returns an
uppercase copy.

### Example 2

```javascript
let csvLine = "apple,banana,cherry";
let fruits = csvLine.split(",");
console.log(fruits);
console.log(fruits.join(" - "));
```

`split` turns a string into an array, and `join` combines an array back
into a string using the given separator.

### Example 3

```javascript
let name = "Ada";
let score = 97.456;
console.log(`${name} scored ${score.toFixed(1)} points`);
```

A template literal embeds expressions directly using `${}`, and
`toFixed(1)` formats the number with one decimal place.

## Common Mistakes

```javascript
let text = "hello";
text[0] = "H";
console.log(text);
```

This silently does nothing, still printing `"hello"`, because strings are
immutable and assigning to an index has no effect. Build a new string
instead, such as `"H" + text.slice(1)`.

## Best Practices

Prefer template literals over string concatenation with `+` for
readability. Use `includes()`, `startsWith()`, and `endsWith()` instead of
manually searching with `indexOf()` when only a boolean result is needed.

## Related Topics

Arrays, Regular Expressions, Template Literals

# Arrays

## Overview

An array is an ordered, mutable list of values. JavaScript arrays can hold
mixed types and provide many built-in methods for transforming their
contents.

## Key Concepts

Arrays are zero-indexed. Many array methods, such as `map`, `filter`, and
`reduce`, do not modify the original array and instead return a new one,
while methods like `push` and `sort` modify the array in place.

## Syntax

```javascript
let numbers = [1, 2, 3];
let mixed = [1, "two", 3.0, true];
let empty = [];
```

## Parameters

Key array methods:

- `array.push(item)`: adds an item to the end, modifying the array.
- `array.map(callback)`: returns a new array with the callback applied to
  every item.
- `array.filter(callback)`: returns a new array containing only items
  where the callback returns true.
- `array.reduce(callback, initialValue)`: combines all items into a single
  value.

## Examples

### Example 1

```javascript
let fruits = ["apple", "banana"];
fruits.push("cherry");
console.log(fruits);
```

`push` adds a single item to the end of the array.

### Example 2

```javascript
let numbers = [1, 2, 3, 4, 5];
let doubled = numbers.map((n) => n * 2);
let evens = numbers.filter((n) => n % 2 === 0);
console.log(doubled);
console.log(evens);
```

`map` transforms every item, and `filter` keeps only items matching the
condition; neither changes the original `numbers` array.

### Example 3

```javascript
let numbers = [1, 2, 3, 4];
let total = numbers.reduce((accumulator, current) => accumulator + current, 0);
console.log(total);
```

`reduce` carries an accumulator through the array, starting from the
initial value `0`, to produce a single combined result.

## Common Mistakes

```javascript
let numbers = [3, 1, 10, 2];
numbers.sort();
console.log(numbers);
```

This prints `[1, 10, 2, 3]` instead of numeric order, because the default
`sort()` compares items as strings. Pass a comparator function, such as
`numbers.sort((a, b) => a - b)`, to sort numbers correctly.

## Best Practices

Prefer `map`, `filter`, and `reduce` over manual `for` loops for
transforming data, since they are more concise and avoid accidental
mutation. Always provide a comparator to `sort()` when working with
numbers.

Objects, Loops, Destructuring

# Objects

## Overview

An object stores a collection of key-value pairs, called properties.
Objects are the foundation for representing structured data in
JavaScript.

## Key Concepts

Property keys are usually strings (or symbols), and values can be any
type, including functions. Objects can be created with object literals,
with the `new` keyword, or with `Object.create()`.

## Syntax

```javascript
let person = { name: "Alice", age: 30 };
let empty = {};
```

## Examples

### Example 1

```javascript
let person = { name: "Alice", age: 30 };
console.log(person.name);
person.age = 31;
console.log(person);
```

Dot notation both reads and writes property values.

### Example 2

```javascript
let person = { name: "Alice", age: 30 };
let key = "age";
console.log(person[key]);
```

Bracket notation allows accessing a property using a variable holding the
property name, which dot notation cannot do.

### Example 3

```javascript
let person = { name: "Alice", city: "Paris" };
for (const [key, value] of Object.entries(person)) {
  console.log(key, "->", value);
}
```

`Object.entries` returns an array of `[key, value]` pairs, which can be
looped over using destructuring inside a `for...of` loop.

## Common Mistakes

```javascript
let data = { name: "Bob" };
console.log(data.age.toString());
```

This raises `TypeError: Cannot read properties of undefined`, because
`data.age` is `undefined`, and calling `.toString()` on `undefined` fails.
Check for the property first, or use optional chaining: `data.age?.toString()`.

## Best Practices

Use object destructuring to extract multiple properties concisely, and
use optional chaining (`?.`) when a property might not exist, to avoid
manual existence checks everywhere.

# Operators

## Overview

Operators perform computations on values: arithmetic, comparison,
logical, and assignment operations.

## Key Concepts

JavaScript has both loose equality (`==`), which performs type coercion,
and strict equality (`===`), which does not. The logical operators `&&`
and `||` can also be used for short-circuit evaluation and default
values, and `??` provides a default only when a value is `null` or
`undefined`.

## Syntax

```javascript
let sum = 2 + 3 * 4;
let isEqual = 5 === 5;
let combined = true && !false;
```

## Examples

### Example 1

```javascript
console.log(5 == "5");
console.log(5 === "5");
```

`==` returns `true` because it converts the string to a number before
comparing, while `===` returns `false` because the types differ.

### Example 2

```javascript
let username = "";
let displayName = username || "Guest";
console.log(displayName);
```

`||` returns the first truthy value, so an empty string falls through to
`"Guest"`.

### Example 3

```javascript
let settings = { volume: 0 };
let volume = settings.volume ?? 50;
console.log(volume);
```

`??` only falls back when the value is `null` or `undefined`, so a
legitimate `0` is preserved here, unlike `||`, which would have treated
`0` as falsy and replaced it.

## Common Mistakes

```javascript
let count = 0;
let displayCount = count || "No items";
console.log(displayCount);
```

This prints `"No items"` even though `count` is a valid value of `0`,
because `0` is falsy. Use `??` instead of `||` when zero or an empty
string should be treated as a valid value.

## Best Practices

Always use `===` and `!==` instead of `==` and `!=` to avoid confusing
type coercion. Use `??` instead of `||` when defaulting a value that
could legitimately be `0`, `""`, or `false`.

## Related Topics

Booleans, Control Flow, Optional Chaining

# Control Flow: if, switch

## Overview

Conditional statements let a program branch between different blocks of
code depending on whether conditions are true.

## Key Concepts

JavaScript evaluates `if/else if/else` chains top to bottom, running the
first matching branch. `switch` compares one value against several
possible cases, and requires `break` to prevent falling through into the
next case.

## Syntax

```javascript
if (condition1) {
  // ...
} else if (condition2) {
  // ...
} else {
  // ...
}
```

```javascript
switch (value) {
  case 1:
    // ...
    break;
  default:
  // ...
}
```

## Examples

### Example 1

```javascript
let score = 82;
let grade;
if (score >= 90) {
  grade = "A";
} else if (score >= 80) {
  grade = "B";
} else {
  grade = "F";
}
console.log(grade);
```

Only the first true condition runs, so `grade` becomes `"B"`.

### Example 2

```javascript
function describeDay(day) {
  switch (day) {
    case "Sat":
    case "Sun":
      return "Weekend";
    default:
      return "Weekday";
  }
}
console.log(describeDay("Sun"));
```

Two `case` labels sharing one block let both `"Sat"` and `"Sun"` produce
the same result.

### Example 3

```javascript
let value = 2;
switch (value) {
  case 1:
    console.log("one");
  case 2:
    console.log("two");
  case 3:
    console.log("three");
    break;
  default:
    console.log("other");
}
```

Without a `break` after `case 2`, execution "falls through" into
`case 3` as well, printing both `"two"` and `"three"`.

## Common Mistakes

```javascript
let value = 1;
switch (value) {
  case 1:
    console.log("one");
  case 2:
    console.log("two");
}
```

Forgetting `break` causes both cases to run even though only `case 1`
matched, since JavaScript falls through by default unless told to stop.

## Best Practices

Add a `break` after every `case` unless fall-through is intentional and
clearly commented. Prefer `if/else` over `switch` when conditions involve
ranges or complex boolean logic rather than exact value matches.

# Loops: for, while, for...of, for...in

## Overview

Loops repeat a block of code, either a fixed number of times, over the
items of a collection, or while a condition remains true.

## Key Concepts

`for...of` iterates over the values of an iterable, such as an array or
string. `for...in` iterates over the enumerable property names of an
object. `break` exits a loop early, and `continue` skips to the next
iteration.

## Syntax

```javascript
for (let i = 0; i < 5; i++) {
  // ...
}

for (const item of iterable) {
  // ...
}

while (condition) {
  // ...
}
```

## Examples

### Example 1

```javascript
for (let i = 0; i < 5; i++) {
  console.log(i);
}
```

A classic counting loop prints the numbers 0 through 4.

### Example 2

```javascript
let fruits = ["apple", "banana", "cherry"];
for (const fruit of fruits) {
  console.log(fruit);
}
```

`for...of` gives direct access to each value, without needing to manage
an index manually.

### Example 3

```javascript
let person = { name: "Ada", age: 28 };
for (const key in person) {
  console.log(key, "->", person[key]);
}
```

`for...in` iterates over property names, and bracket notation is used to
look up each value.

## Common Mistakes

```javascript
let numbers = [10, 20, 30];
for (const index in numbers) {
  console.log(index);
}
```

This logs the string indices `"0"`, `"1"`, `"2"` instead of the values,
because `for...in` on an array iterates over its indices, not its values.
Use `for...of` when the values themselves are needed.

## Best Practices

Prefer `for...of` for iterating over array or string values, and
`Object.entries` combined with `for...of` for iterating over an object's
key-value pairs. Avoid `for...in` on arrays.

Arrays, Objects, Iterators

# Functions, Arrow Functions, and Closures

## Overview

Functions are reusable blocks of code. JavaScript supports several ways
to define them, including traditional function declarations and the more
compact arrow function syntax.

## Key Concepts

A closure is a function that "remembers" the variables from the scope in
which it was created, even after that outer scope has finished executing.
Arrow functions do not have their own `this`, and instead inherit `this`
from the surrounding scope, which is different from regular functions.

## Syntax

```javascript
function add(a, b) {
  return a + b;
}

const multiply = (a, b) => a * b;
```

## Examples

### Example 1

```javascript
function greet(name, greeting = "Hello") {
  return `${greeting}, ${name}!`;
}
console.log(greet("World"));
```

Default parameter values are used when an argument is not provided.

### Example 2

```javascript
function makeCounter() {
  let count = 0;
  return function () {
    count += 1;
    return count;
  };
}

const counter = makeCounter();
console.log(counter());
console.log(counter());
```

The inner function forms a closure over `count`, remembering its value
between calls even though `makeCounter` has already returned.

### Example 3

```javascript
const numbers = [1, 2, 3, 4, 5];
const squares = numbers.map((n) => n * n);
console.log(squares);
```

Arrow functions are commonly used as short callbacks passed to array
methods like `map`.

## Common Mistakes

```javascript
const person = {
  name: "Ada",
  greet: () => {
    console.log(`Hello, ${this.name}`);
  },
};
person.greet();
```

This prints `"Hello, undefined"`, because arrow functions do not bind
their own `this`; here `this` refers to the outer scope, not `person`.
Use a regular function (`greet() { ... }`) when `this` should refer to the
object itself.

## Best Practices

Use arrow functions for short callbacks, especially where inheriting
`this` from the surrounding scope is desired. Use regular function syntax
for object methods that need their own `this`.

Scope, Classes, this keyword

# Classes and OOP

## Overview

JavaScript classes provide syntax for creating objects with shared
behavior, built on top of the language's underlying prototype system.

## Key Concepts

A class defines a constructor and methods. `extends` lets one class
inherit from another, and `super` calls the parent class's constructor or
methods. Fields and methods can be marked `static` to belong to the class
itself, rather than to instances.

## Syntax

```javascript
class Dog {
  constructor(name) {
    this.name = name;
  }

  bark() {
    return `${this.name} says woof!`;
  }
}
```

## Examples

### Example 1

```javascript
class Dog {
  constructor(name) {
    this.name = name;
  }

  bark() {
    return `${this.name} says woof!`;
  }
}

const myDog = new Dog("Rex");
console.log(myDog.bark());
```

`new Dog("Rex")` calls the constructor, setting up `this.name` for the
new instance.

### Example 2

```javascript
class Animal {
  speak() {
    return "...";
  }
}

class Cat extends Animal {
  speak() {
    return "Meow";
  }
}

const animals = [new Animal(), new Cat()];
for (const creature of animals) {
  console.log(creature.speak());
}
```

`Cat extends Animal` and overrides `speak`, demonstrating polymorphism:
the same method call produces different results based on the actual
class.

### Example 3

```javascript
class Counter {
  #count = 0;

  increment() {
    this.#count += 1;
  }

  get value() {
    return this.#count;
  }
}

const counter = new Counter();
counter.increment();
counter.increment();
console.log(counter.value);
```

The `#count` field is private, only accessible inside the class, and
`get value()` defines a computed property that reads like a plain
attribute from the outside.

## Common Mistakes

```javascript
class Point {
  constructor(x, y) {
    this.x = x;
    this.y = y;
  }
}

const p = Point(1, 2);
```

Calling a class without `new` raises `TypeError: Class constructor Point
cannot be invoked without 'new'`, because classes must always be
instantiated with `new`.

## Best Practices

Use private fields (`#field`) for internal state that should not be
accessed from outside the class. Prefer composition over deep inheritance
chains when a "has-a" relationship fits the problem better than "is-a".

Objects, Inheritance, Encapsulation

# Error Handling

## Overview

Error handling allows a JavaScript program to respond to unexpected
failures, such as invalid input or a failed network request, without
crashing entirely.

## Key Concepts

`try` contains code that might throw an error. `catch` handles the error
if one occurs. `finally` always runs, whether or not an error occurred.
Custom error types can be created by extending the built-in `Error` class.

## Syntax

```javascript
try {
  riskyOperation();
} catch (error) {
  handleError(error);
} finally {
  cleanup();
}
```

## Examples

### Example 1

```javascript
function parseAge(text) {
  const age = Number(text);
  if (Number.isNaN(age)) {
    throw new Error("Invalid age");
  }
  return age;
}

try {
  console.log(parseAge("abc"));
} catch (error) {
  console.log("Failed:", error.message);
}
```

`throw new Error(...)` raises an exception, and `catch` receives it,
allowing a friendly message to be shown instead of crashing.

### Example 2

```javascript
class InsufficientFundsError extends Error {
  constructor(message) {
    super(message);
    this.name = "InsufficientFundsError";
  }
}

function withdraw(balance, amount) {
  if (amount > balance) {
    throw new InsufficientFundsError("Not enough funds");
  }
  return balance - amount;
}

try {
  withdraw(50, 100);
} catch (error) {
  console.log(error.name, "-", error.message);
}
```

Extending `Error` creates a custom error type that still behaves like a
normal error, including having a `.message` and a stack trace.

### Example 3

```javascript
async function loadData() {
  try {
    const response = await fetch("https://example.com/data");
    return await response.json();
  } catch (error) {
    console.log("Request failed:", error.message);
    return null;
  } finally {
    console.log("Request attempt finished");
  }
}
```

`finally` runs regardless of whether the request succeeded or failed,
making it a good place for cleanup logic like hiding a loading indicator.

## Common Mistakes

```javascript
try {
  JSON.parse("{ invalid json");
} catch (e) {}
```

Silently swallowing an error with an empty `catch` block hides bugs; at
minimum, log the error so problems are visible during development.

## Best Practices

Catch errors as close as possible to where you can meaningfully handle
them, and always include useful information when re-throwing or logging.
Prefer custom error classes over throwing plain strings, since `Error`
objects carry a stack trace.

## Related Topics

Custom Exceptions, Promises, Async/Await

# Promises and Async/Await

## Overview

Promises represent a value that may not be available yet, such as the
result of a network request. `async`/`await` is syntax built on top of
promises that makes asynchronous code read like ordinary synchronous
code.

## Key Concepts

A promise is in one of three states: pending, fulfilled, or rejected.
`.then()` handles a fulfilled promise, and `.catch()` handles a rejected
one. An `async` function always returns a promise, and `await` pauses
execution inside that function until the awaited promise settles.

## Syntax

```javascript
fetchData()
  .then((result) => console.log(result))
  .catch((error) => console.log(error));

async function run() {
  const result = await fetchData();
  console.log(result);
}
```

## Examples

### Example 1

```javascript
function wait(ms) {
  return new Promise((resolve) => setTimeout(resolve, ms));
}

wait(1000).then(() => console.log("One second passed"));
```

`new Promise` wraps `setTimeout`, resolving after the given delay; `.then`
runs once the promise resolves.

### Example 2

```javascript
async function getUser(id) {
  const response = await fetch(`https://example.com/users/${id}`);
  if (!response.ok) {
    throw new Error("User not found");
  }
  return response.json();
}

async function main() {
  try {
    const user = await getUser(1);
    console.log(user);
  } catch (error) {
    console.log("Error:", error.message);
  }
}
```

`await` pauses the function until each promise resolves, and errors from
either `fetch` or the manual `throw` are caught by the surrounding
`try/catch`.

### Example 3

```javascript
async function loadAll() {
  const [users, posts] = await Promise.all([
    fetch("https://example.com/users").then((r) => r.json()),
    fetch("https://example.com/posts").then((r) => r.json()),
  ]);
  console.log(users, posts);
}
```

`Promise.all` runs multiple promises concurrently and waits for all of
them to finish, which is faster than awaiting them one at a time when
they do not depend on each other.

## Common Mistakes

```javascript
async function getData() {
  return 42;
}

const result = getData();
console.log(result);
```

This prints a `Promise` object, not `42`, because an `async` function
always returns a promise; the caller must `await` it or use `.then()` to
get the actual value.

## Best Practices

Prefer `async/await` over chained `.then()` calls for readability, but
still wrap awaited code in `try/catch` to handle rejections. Use
`Promise.all` when multiple independent asynchronous operations can run
concurrently instead of sequentially.

## Related Topics

Error Handling, Functions, Event Loop

# JSON

## Overview

`JSON` is a global object providing methods to convert between JavaScript
values and JSON text, commonly used for sending and receiving data over a
network.

## Key Concepts

`JSON.stringify` converts a JavaScript value into a JSON string.
`JSON.parse` converts a JSON string back into a JavaScript value. Not
every JavaScript value can be represented in JSON; functions and
`undefined` are simply omitted.

## Syntax

```javascript
const text = JSON.stringify({ name: "Ada" });
const data = JSON.parse(text);
```

## Examples

### Example 1

```javascript
const person = { name: "Ada", age: 28, active: true };
const text = JSON.stringify(person, null, 2);
console.log(text);
```

The third argument, `2`, tells `stringify` to indent the output with two
spaces, producing readable, formatted JSON text.

### Example 2

```javascript
const text = '{"name": "Bob", "scores": [90, 85, 88]}';
const data = JSON.parse(text);
console.log(data.name, data.scores);
```

`JSON.parse` converts a JSON object into a plain JavaScript object, and a
JSON array into a JavaScript array.

### Example 3

```javascript
const user = { name: "Ada", password: "secret", email: "ada@example.com" };
const publicText = JSON.stringify(user, ["name", "email"]);
console.log(publicText);
```

Passing an array as the second argument to `stringify` limits the output
to only the listed properties, which is useful for hiding sensitive
fields.

## Common Mistakes

```javascript
const data = { created: new Date() };
console.log(JSON.stringify(data));
```

This works, but silently converts the `Date` into an ISO string, which is
not automatically converted back into a `Date` object when later parsed
with `JSON.parse`; the result stays a plain string unless manually
converted back.

## Best Practices

Wrap `JSON.parse` in a `try/catch` block when working with data from an
external or untrusted source, since malformed JSON throws a
`SyntaxError`. Avoid storing functions or circular references in objects
that will be serialized with `JSON.stringify`.

## Related Topics

Objects, Error Handling, fetch API

# Destructuring and the Spread Operator

## Overview

Destructuring extracts values from arrays or objects into separate
variables in a single statement. The spread operator expands an iterable
or object into individual elements or properties.

## Key Concepts

Object destructuring matches by property name; array destructuring
matches by position. The spread operator (`...`) can copy arrays and
objects, merge them, or collect remaining function arguments.

## Syntax

```javascript
const { name, age } = person;
const [first, second] = numbers;
const combined = [...arrayOne, ...arrayTwo];
```

## Examples

### Example 1

```javascript
const person = { name: "Ada", age: 28, city: "London" };
const { name, city } = person;
console.log(name, city);
```

Object destructuring pulls out only the named properties into their own
variables, in this case `name` and `city`.

### Example 2

```javascript
const numbers = [1, 2, 3, 4, 5];
const [first, second, ...rest] = numbers;
console.log(first, second, rest);
```

Array destructuring assigns the first two values by position, and `...rest`
gathers everything remaining into a new array.

### Example 3

```javascript
function sum(...numbers) {
  return numbers.reduce((total, n) => total + n, 0);
}
console.log(sum(1, 2, 3, 4));

const defaults = { theme: "light", fontSize: 14 };
const userSettings = { fontSize: 18 };
const merged = { ...defaults, ...userSettings };
console.log(merged);
```

`...numbers` inside a function definition collects all arguments into an
array, while spreading two objects merges their properties, with later
objects overriding earlier ones for shared keys.

## Common Mistakes

```javascript
const [a, b] = { 0: "x", 1: "y" };
```

This raises `TypeError` because array destructuring requires an iterable,
and a plain object is not iterable even if it has numeric-looking keys.
Use object destructuring, or convert to an actual array first.

## Best Practices

Use destructuring in function parameters to make it clear which
properties of an object argument are actually used. Use the spread
operator instead of `Object.assign` for merging objects, since it is more
concise and widely supported in modern code.

## Related Topics

Arrays, Objects, Functions

# Variables and Data Types

## Overview

Java is a statically typed language, meaning every variable's type must
be declared and is checked at compile time, before the program runs.

## Key Concepts

Java has eight primitive types: `byte`, `short`, `int`, `long`, `float`,
`double`, `char`, and `boolean`. Everything else, including `String` and
arrays, is a reference type, meaning the variable holds a reference to an
object rather than the value itself.

## Syntax

```java
int age = 30;
double price = 19.99;
boolean isActive = true;
String name = "Ada";
```

## Examples

### Example 1

```java
public class Main {
    public static void main(String[] args) {
        int count = 10;
        count = count + 5;
        System.out.println(count);
    }
}
```

The variable `count` is declared as `int` and reassigned, and Java checks
at compile time that only integer-compatible values can be assigned to
it.

### Example 2

```java
public class Main {
    public static void main(String[] args) {
        final double TAX_RATE = 0.08;
        double price = 100.0;
        double total = price + (price * TAX_RATE);
        System.out.println(total);
    }
}
```

`final` marks a variable as a constant; attempting to reassign it later
would cause a compile-time error.

### Example 3

```java
public class Main {
    public static void main(String[] args) {
        int wholeNumber = 7;
        double result = wholeNumber / 2;
        double correctResult = wholeNumber / 2.0;
        System.out.println(result);
        System.out.println(correctResult);
    }
}
```

Dividing two `int` values performs integer division, discarding the
remainder, even when the result is assigned to a `double`; at least one
operand must be a floating-point type to get a fractional result.

## Common Mistakes

```java
int age;
System.out.println(age);
```

This causes a compile-time error, "variable age might not have been
initialized," because Java requires local variables to be assigned a
value before they are used, unlike some other languages that default to
zero automatically.

## Best Practices

Choose the narrowest type that comfortably fits the expected range of
values, such as `int` for everyday whole numbers rather than `long`
unless very large values are expected. Mark variables `final` whenever
they should not be reassigned, to make that intent clear and catch
mistakes early.

## Related Topics

Operators, Strings, Type Casting

# Strings

## Overview

`String` in Java represents a sequence of characters and is one of the
most commonly used reference types, backed by a rich set of built-in
methods.

## Key Concepts

`String` objects are immutable in Java; every method that appears to
modify a string actually returns a new `String` object. For building
strings through many modifications, `StringBuilder` is more efficient
than repeated string concatenation.

## Syntax

```java
String greeting = "Hello";
String name = "World";
String message = greeting + ", " + name + "!";
```

## Examples

### Example 1

```java
public class Main {
    public static void main(String[] args) {
        String text = "  Java is powerful  ";
        System.out.println(text.trim());
        System.out.println(text.trim().toUpperCase());
    }
}
```

`trim()` removes surrounding whitespace, and `toUpperCase()` returns a
new, uppercase string.

### Example 2

```java
public class Main {
    public static void main(String[] args) {
        String csvLine = "apple,banana,cherry";
        String[] fruits = csvLine.split(",");
        for (String fruit : fruits) {
            System.out.println(fruit);
        }
    }
}
```

`split` divides a string into an array of substrings based on the given
delimiter.

### Example 3

```java
public class Main {
    public static void main(String[] args) {
        StringBuilder builder = new StringBuilder();
        for (int i = 1; i <= 5; i++) {
            builder.append(i).append(", ");
        }
        System.out.println(builder.toString());
    }
}
```

`StringBuilder` accumulates text efficiently across a loop, since it
modifies an internal buffer instead of creating a new `String` on every
step.

## Common Mistakes

```java
String a = "hello";
String b = "hello";
String c = new String("hello");
System.out.println(a == b);
System.out.println(a == c);
```

`a == b` is `true` because Java reuses identical string literals from an
internal pool, but `a == c` is `false` because `new String(...)` always
creates a distinct object; comparing string contents should always use
`.equals()`, not `==`.

## Best Practices

Always compare string contents with `.equals()` rather than `==`. Use
`StringBuilder` instead of repeated `+` concatenation inside loops, since
each `+` on plain strings creates a new object.

## Related Topics

Arrays, StringBuilder, Regular Expressions

# Arrays

## Overview

An array in Java is a fixed-size, ordered collection of elements of the
same type.

## Key Concepts

Once created, an array's length cannot change; a new array must be
created if more space is needed. Arrays are zero-indexed, and accessing
an index outside the valid range throws
`ArrayIndexOutOfBoundsException` at runtime.

## Syntax

```java
int[] numbers = {1, 2, 3};
String[] names = new String[3];
```

## Examples

### Example 1

```java
public class Main {
    public static void main(String[] args) {
        int[] numbers = {10, 20, 30};
        for (int i = 0; i < numbers.length; i++) {
            System.out.println(numbers[i]);
        }
    }
}
```

`.length` gives the size of the array, commonly used to bound a loop over
its indices.

### Example 2

```java
public class Main {
    public static void main(String[] args) {
        int[] numbers = {5, 3, 8, 1};
        java.util.Arrays.sort(numbers);
        System.out.println(java.util.Arrays.toString(numbers));
    }
}
```

`Arrays.sort` sorts the array in place, and `Arrays.toString` produces a
readable representation, since printing an array directly shows its
memory reference instead of its contents.

### Example 3

```java
public class Main {
    public static void main(String[] args) {
        int[][] grid = {
            {1, 2, 3},
            {4, 5, 6}
        };
        for (int[] row : grid) {
            for (int value : row) {
                System.out.print(value + " ");
            }
            System.out.println();
        }
    }
}
```

A two-dimensional array is an array of arrays, and nested `for-each` loops
visit every element.

## Common Mistakes

```java
int[] numbers = {1, 2, 3};
System.out.println(numbers[3]);
```

This throws `ArrayIndexOutOfBoundsException`, because valid indices for a
three-element array are only 0, 1, and 2; there is no index 3.

## Best Practices

Use enhanced `for` loops (`for (int value : numbers)`) when the index
itself is not needed, since it is safer and more concise. Prefer
`ArrayList` over a plain array when the collection's size needs to change
after creation.

Collections, Loops, ArrayList

# Operators

## Overview

Java operators perform arithmetic, comparison, logical, and assignment
operations, similar in spirit to most C-family languages.

## Key Concepts

`==` compares primitive values directly, but for reference types like
`String`, it compares object identity, not content, which is why
`.equals()` is used for content comparison. Java also has increment
(`++`) and decrement (`--`) operators, and compound assignment operators
like `+=`.

## Syntax

```java
int sum = 2 + 3 * 4;
boolean isEqual = (5 == 5);
boolean combined = true && !false;
```

## Examples

### Example 1

```java
public class Main {
    public static void main(String[] args) {
        int a = 10, b = 3;
        System.out.println(a + b);
        System.out.println(a / b);
        System.out.println(a % b);
    }
}
```

Integer division `a / b` truncates toward zero, and `%` gives the
remainder.

### Example 2

```java
public class Main {
    public static void main(String[] args) {
        int score = 85;
        boolean passed = score >= 60;
        boolean withHonors = score >= 90 && passed;
        System.out.println(passed);
        System.out.println(withHonors);
    }
}
```

Comparison operators produce `boolean` results that can be combined with
`&&` and `||`.

### Example 3

```java
public class Main {
    public static void main(String[] args) {
        int counter = 0;
        counter++;
        counter += 5;
        System.out.println(counter);
    }
}
```

`++` increments by one, and `+=` adds a value and assigns the result back
in a single step.

## Common Mistakes

```java
String a = "hello";
String b = "hel" + "lo";
String c = new String("hello");
System.out.println(a == c);
```

Beginners often expect `==` to compare string content in Java the way
some other languages do, but it compares references; `a == c` is `false`
even though the text is identical, because `new String(...)` produces a
separate object.

## Best Practices

Always use `.equals()` for comparing the content of objects, including
`String`, and reserve `==` for primitives or for intentionally checking
whether two references point to the exact same object.

## Related Topics

Booleans, Strings, Control Flow

# Control Flow: if, switch

## Overview

Conditional statements let a Java program choose between different
branches of execution based on boolean conditions.

## Key Concepts

`if/else if/else` chains run the first branch whose condition is true.
Modern Java (14 and later) supports a more concise "switch expression"
syntax using arrows, in addition to the traditional colon-based `switch`
statement.

## Syntax

```java
if (condition1) {
    // ...
} else if (condition2) {
    // ...
} else {
    // ...
}
```

```java
switch (value) {
    case 1 -> System.out.println("one");
    case 2 -> System.out.println("two");
    default -> System.out.println("other");
}
```

## Examples

### Example 1

```java
public class Main {
    public static void main(String[] args) {
        int score = 82;
        String grade;
        if (score >= 90) {
            grade = "A";
        } else if (score >= 80) {
            grade = "B";
        } else {
            grade = "F";
        }
        System.out.println(grade);
    }
}
```

Only the first true branch runs, assigning `"B"` to `grade`.

### Example 2

```java
public class Main {
    public static void main(String[] args) {
        String day = "SAT";
        String type = switch (day) {
            case "SAT", "SUN" -> "Weekend";
            default -> "Weekday";
        };
        System.out.println(type);
    }
}
```

The arrow-based `switch` expression can directly produce a value, and
multiple labels can share one arrow branch.

### Example 3

```java
public class Main {
    public static void main(String[] args) {
        int value = 2;
        switch (value) {
            case 1:
                System.out.println("one");
                break;
            case 2:
                System.out.println("two");
                break;
            default:
                System.out.println("other");
        }
    }
}
```

The traditional colon-based `switch` requires `break` after each case to
prevent fall-through into the next one.

## Common Mistakes

```java
int value = 1;
switch (value) {
    case 1:
        System.out.println("one");
    case 2:
        System.out.println("two");
}
```

Forgetting `break` in a traditional `switch` statement causes execution
to "fall through" into the next case, printing both `"one"` and `"two"`
even though only `case 1` matched.

## Best Practices

Prefer the modern arrow-based `switch` expression when targeting Java 14
or later, since it removes the need for `break` and avoids accidental
fall-through entirely.

## Related Topics

Operators, Loops, Enums

# Loops: for, while, for-each

## Overview

Loops let a Java program repeat a block of code a fixed number of times,
while a condition holds, or once for every element of a collection.

## Key Concepts

The enhanced `for` loop, often called a for-each loop, iterates directly
over the elements of an array or a `Collection`, without manual index
management. `break` exits a loop immediately, and `continue` skips to the
next iteration.

## Syntax

```java
for (int i = 0; i < 5; i++) {
    // ...
}

for (String item : items) {
    // ...
}

while (condition) {
    // ...
}
```

## Examples

### Example 1

```java
public class Main {
    public static void main(String[] args) {
        for (int i = 0; i < 5; i++) {
            System.out.println(i);
        }
    }
}
```

A classic counting loop prints the numbers 0 through 4.

### Example 2

```java
import java.util.List;

public class Main {
    public static void main(String[] args) {
        List<String> fruits = List.of("apple", "banana", "cherry");
        for (String fruit : fruits) {
            System.out.println(fruit);
        }
    }
}
```

The for-each loop reads naturally as "for each fruit in fruits," without
needing an explicit index variable.

### Example 3

```java
public class Main {
    public static void main(String[] args) {
        int count = 0;
        while (count < 10) {
            if (count % 2 == 0) {
                count++;
                continue;
            }
            System.out.println(count);
            count++;
        }
    }
}
```

`continue` skips the `println` for even numbers, so only odd numbers less
than 10 are printed.

## Common Mistakes

```java
int count = 0;
while (count < 5) {
    System.out.println(count);
}
```

This is an infinite loop because `count` is never updated inside the
loop body, so the condition `count < 5` never becomes false.

## Best Practices

Use the enhanced for-each loop whenever the index itself is not needed,
since it is less error-prone than manually managing an index variable.
Use a traditional `for` loop when the index or a non-standard step is
actually required.

## Related Topics

Arrays, Collections, Recursion

# Methods

## Overview

A method is a named, reusable block of code defined inside a class,
similar in purpose to a function in other languages, but always
associated with a class.

## Key Concepts

Every method has a declared return type (or `void` if it returns
nothing), a name, and a parameter list. Methods can be overloaded, giving
the same name to multiple methods as long as their parameter lists
differ.

## Syntax

```java
public int add(int a, int b) {
    return a + b;
}
```

## Parameters

- The return type appears before the method name, and must match what
  `return` produces, or be `void` if nothing is returned.
- Parameters are declared with an explicit type for each one.
- `static` methods belong to the class itself and can be called without
  creating an instance.

## Return Value

A method declared with a return type other than `void` must return a
value of a compatible type on every possible execution path, or the code
will not compile.

## Examples

### Example 1

```java
public class Calculator {
    public int add(int a, int b) {
        return a + b;
    }
}

public class Main {
    public static void main(String[] args) {
        Calculator calculator = new Calculator();
        System.out.println(calculator.add(3, 4));
    }
}
```

`add` takes two `int` parameters and returns their sum as an `int`.

### Example 2

```java
public class Printer {
    public void printLine(String text) {
        System.out.println(text);
    }
}
```

A `void` method performs an action but does not produce a return value.

### Example 3

```java
public class MathUtils {
    public static int square(int value) {
        return value * value;
    }

    public static double square(double value) {
        return value * value;
    }
}

public class Main {
    public static void main(String[] args) {
        System.out.println(MathUtils.square(4));
        System.out.println(MathUtils.square(4.5));
    }
}
```

Method overloading allows `square` to work with both `int` and `double`
arguments, with Java choosing the correct version based on the argument
type.

## Common Mistakes

```java
public int multiply(int a, int b) {
    int result = a * b;
}
```

This is a compile-time error, "missing return statement," because a
method declared to return `int` must actually return a value on every
path through the method.

## Best Practices

Keep methods focused on a single responsibility, and choose descriptive
names that make it clear what the method does. Prefer returning early for
edge cases at the top of a method, rather than deeply nesting the main
logic inside multiple conditions.

## Related Topics

Classes, Overloading, Static Members

# Classes, Objects, and Constructors

## Overview

A class is a blueprint for creating objects, describing the fields
(data) and methods (behavior) that every instance of that class will
have.

## Key Concepts

A constructor is a special method, matching the class name, that runs
when an object is created with `new`. If no constructor is written, Java
provides a default no-argument constructor automatically, but that
default disappears once any constructor is written explicitly.

## Syntax

```java
public class Dog {
    private String name;

    public Dog(String name) {
        this.name = name;
    }

    public String bark() {
        return name + " says woof!";
    }
}
```

## Examples

### Example 1

```java
public class Dog {
    private String name;

    public Dog(String name) {
        this.name = name;
    }

    public String bark() {
        return name + " says woof!";
    }
}

public class Main {
    public static void main(String[] args) {
        Dog myDog = new Dog("Rex");
        System.out.println(myDog.bark());
    }
}
```

`new Dog("Rex")` calls the constructor, which sets the private `name`
field for that specific instance.

### Example 2

```java
public class Point {
    private int x;
    private int y;

    public Point() {
        this(0, 0);
    }

    public Point(int x, int y) {
        this.x = x;
        this.y = y;
    }

    public String toString() {
        return "(" + x + ", " + y + ")";
    }
}
```

`this(0, 0)` calls another constructor in the same class, avoiding
duplicated initialization logic between the no-argument and two-argument
constructors.

### Example 3

```java
public class BankAccount {
    private double balance;

    public BankAccount(double initialBalance) {
        balance = initialBalance;
    }

    public void deposit(double amount) {
        balance += amount;
    }

    public double getBalance() {
        return balance;
    }
}

public class Main {
    public static void main(String[] args) {
        BankAccount account = new BankAccount(100.0);
        account.deposit(50.0);
        System.out.println(account.getBalance());
    }
}
```

The `private` field `balance` can only be changed through the class's own
methods, keeping the account's internal state controlled.

## Common Mistakes

```java
public class Point {
    int x;
    int y;

    public void Point(int x, int y) {
        this.x = x;
        this.y = y;
    }
}
```

Adding a return type (`void`) before the method name turns this into an
ordinary method named `Point`, not a constructor; a real constructor must
have no return type at all, not even `void`.

## Best Practices

Keep fields `private` and expose controlled access through methods
(often called getters and setters), rather than making fields `public`
directly. Use constructor chaining with `this(...)` to avoid repeating
initialization logic across multiple constructors.

## Related Topics

Encapsulation, Inheritance, this keyword

# Inheritance and Polymorphism

## Overview

Inheritance lets one class reuse and extend the fields and methods of
another class. Polymorphism allows the same method call to behave
differently depending on the actual type of the object at runtime.

## Key Concepts

`extends` establishes an inheritance relationship, and `super` refers to
the parent class, used to call its constructor or an overridden method.
A subclass can override a method from its parent by redefining it with
the same signature, and the `@Override` annotation helps the compiler
catch mistakes in that process.

## Syntax

```java
public class Animal {
    public String speak() {
        return "...";
    }
}

public class Cat extends Animal {
    @Override
    public String speak() {
        return "Meow";
    }
}
```

## Examples

### Example 1

```java
public class Animal {
    public String speak() {
        return "...";
    }
}

public class Cat extends Animal {
    @Override
    public String speak() {
        return "Meow";
    }
}

public class Main {
    public static void main(String[] args) {
        Animal[] animals = { new Animal(), new Cat() };
        for (Animal creature : animals) {
            System.out.println(creature.speak());
        }
    }
}
```

Even though both items are stored as `Animal`, calling `speak()` on each
uses the actual runtime type's version of the method, printing `"..."`
and then `"Meow"`.

### Example 2

```java
public class Vehicle {
    protected String name;

    public Vehicle(String name) {
        this.name = name;
    }

    public String describe() {
        return "Vehicle: " + name;
    }
}

public class Car extends Vehicle {
    public Car(String name) {
        super(name);
    }

    @Override
    public String describe() {
        return "Car: " + super.describe();
    }
}
```

`super(name)` calls the parent class's constructor, and `super.describe()`
calls the parent's version of an overridden method, letting `Car` build
on top of `Vehicle`'s behavior instead of fully replacing it.

### Example 3

```java
public abstract class Shape {
    public abstract double area();

    public String summary() {
        return "Area: " + area();
    }
}

public class Circle extends Shape {
    private double radius;

    public Circle(double radius) {
        this.radius = radius;
    }

    @Override
    public double area() {
        return Math.PI * radius * radius;
    }
}
```

An `abstract` class can declare a method with no body, forcing every
concrete subclass to provide its own implementation, while still sharing
common logic like `summary()`.

## Common Mistakes

```java
public class Animal {
    public String speak() {
        return "...";
    }
}

public class Dog extends Animal {
    public String Speak() {
        return "Woof";
    }
}
```

Capitalizing `Speak` differently from the parent's `speak` does not
override the method; it creates an entirely new, unrelated method, so
calling `speak()` on a `Dog` still returns `"..."`. Use `@Override` so the
compiler flags mistakes like this.

## Best Practices

Always add `@Override` when overriding a method, since it causes a
compile-time error if the signature does not actually match anything in
the parent class. Favor interfaces over abstract classes when multiple,
unrelated classes need to share a contract but not shared implementation.

## Related Topics

Interfaces, Abstract Classes, Encapsulation

# Interfaces and Abstract Classes

## Overview

An interface defines a contract of methods that implementing classes must
provide, without specifying how those methods work. An abstract class is
similar but can also include shared fields and fully implemented methods.

## Key Concepts

A class can implement multiple interfaces, but can only extend one class
(including one abstract class), since Java does not support multiple
inheritance of implementation. Since Java 8, interfaces can also include
`default` methods with a body.

## Syntax

```java
public interface Drawable {
    void draw();
}

public class Circle implements Drawable {
    @Override
    public void draw() {
        System.out.println("Drawing a circle");
    }
}
```

## Examples

### Example 1

```java
public interface Drawable {
    void draw();
}

public class Circle implements Drawable {
    @Override
    public void draw() {
        System.out.println("Drawing a circle");
    }
}

public class Square implements Drawable {
    @Override
    public void draw() {
        System.out.println("Drawing a square");
    }
}

public class Main {
    public static void main(String[] args) {
        Drawable[] shapes = { new Circle(), new Square() };
        for (Drawable shape : shapes) {
            shape.draw();
        }
    }
}
```

Both `Circle` and `Square` implement `Drawable`, so they can be stored
and used interchangeably through the shared interface type.

### Example 2

```java
public interface Greeter {
    String getName();

    default String greet() {
        return "Hello, " + getName() + "!";
    }
}

public class Person implements Greeter {
    private String name;

    public Person(String name) {
        this.name = name;
    }

    @Override
    public String getName() {
        return name;
    }
}
```

A `default` method provides a working implementation directly on the
interface, so implementing classes only need to supply `getName()` to
also gain `greet()` for free.

### Example 3

```java
public abstract class Employee {
    protected String name;

    public Employee(String name) {
        this.name = name;
    }

    public abstract double calculatePay();

    public void printPaycheck() {
        System.out.println(name + ": " + calculatePay());
    }
}

public class SalariedEmployee extends Employee {
    private double monthlySalary;

    public SalariedEmployee(String name, double monthlySalary) {
        super(name);
        this.monthlySalary = monthlySalary;
    }

    @Override
    public double calculatePay() {
        return monthlySalary;
    }
}
```

The abstract class provides shared state (`name`) and behavior
(`printPaycheck`), while forcing each subclass to define its own pay
calculation.

## Common Mistakes

```java
public interface Drawable {
    void draw();
}

public class Circle implements Drawable {
}
```

This is a compile-time error because `Circle` implements `Drawable` but
does not provide a `draw()` method; every non-abstract class implementing
an interface must implement all of its abstract methods.

## Best Practices

Use an interface to describe "can do" capabilities shared across
unrelated classes, and an abstract class when related classes should
share both a contract and some common implementation or state.

## Related Topics

Inheritance, Polymorphism, Generics

# Exception Handling

## Overview

Java uses exceptions to represent errors that occur during program
execution, allowing a program to handle problems gracefully instead of
crashing.

## Key Concepts

Java distinguishes checked exceptions, which the compiler forces the
caller to handle or declare, from unchecked exceptions (subclasses of
`RuntimeException`), which do not require explicit handling.
`try`/`catch`/`finally` blocks are used to catch and respond to
exceptions.

## Syntax

```java
try {
    riskyOperation();
} catch (SomeException e) {
    handleError(e);
} finally {
    cleanup();
}
```

## Examples

### Example 1

```java
public class Main {
    public static void main(String[] args) {
        try {
            int result = 10 / 0;
        } catch (ArithmeticException e) {
            System.out.println("Cannot divide by zero: " + e.getMessage());
        }
    }
}
```

Dividing by zero with integers throws `ArithmeticException`, which is
caught and handled with a friendly message instead of crashing the
program.

### Example 2

```java
public class InsufficientFundsException extends Exception {
    public InsufficientFundsException(String message) {
        super(message);
    }
}

public class BankAccount {
    private double balance;

    public BankAccount(double balance) {
        this.balance = balance;
    }

    public void withdraw(double amount) throws InsufficientFundsException {
        if (amount > balance) {
            throw new InsufficientFundsException("Not enough funds");
        }
        balance -= amount;
    }
}

public class Main {
    public static void main(String[] args) {
        BankAccount account = new BankAccount(50.0);
        try {
            account.withdraw(100.0);
        } catch (InsufficientFundsException e) {
            System.out.println("Withdrawal failed: " + e.getMessage());
        }
    }
}
```

A custom checked exception extends `Exception`, and any method that can
throw it must declare that with `throws`, forcing callers to handle it.

### Example 3

```java
public class Main {
    public static void main(String[] args) {
        try {
            int[] numbers = {1, 2, 3};
            System.out.println(numbers[5]);
        } catch (ArrayIndexOutOfBoundsException e) {
            System.out.println("Invalid index: " + e.getMessage());
        } finally {
            System.out.println("Done attempting access");
        }
    }
}
```

`finally` runs whether or not an exception occurred, making it a
reliable place for cleanup logic such as closing a file or a connection.

## Common Mistakes

```java
try {
    riskyOperation();
} catch (Exception e) {
}
```

An empty `catch` block silently swallows the error, hiding real problems
during development; at minimum, log the exception's message or stack
trace.

## Best Practices

Catch the most specific exception type that makes sense, rather than a
broad `catch (Exception e)`, unless truly handling every possible case
the same way. Use try-with-resources for closable resources like files
and streams, so they are closed automatically even if an exception
occurs.

## Related Topics

Custom Exceptions, try-with-resources, File I/O

# Collections: ArrayList, HashMap, HashSet

## Overview

The Java Collections Framework provides ready-to-use, generic data
structures such as resizable lists, key-value maps, and unique-item sets,
built on top of shared interfaces like `List`, `Map`, and `Set`.

## Key Concepts

`ArrayList` is a resizable array, unlike a plain array whose size is
fixed. `HashMap` stores key-value pairs with fast average-time lookup by
key. `HashSet` stores unique items with fast membership testing, similar
in spirit to a mathematical set.

## Syntax

```java
import java.util.ArrayList;
import java.util.HashMap;
import java.util.HashSet;

ArrayList<String> names = new ArrayList<>();
HashMap<String, Integer> ages = new HashMap<>();
HashSet<String> tags = new HashSet<>();
```

## Examples

### Example 1

```java
import java.util.ArrayList;

public class Main {
    public static void main(String[] args) {
        ArrayList<String> fruits = new ArrayList<>();
        fruits.add("apple");
        fruits.add("banana");
        fruits.remove("apple");
        System.out.println(fruits);
    }
}
```

Unlike a plain array, `ArrayList` can grow and shrink as items are added
or removed.

### Example 2

```java
import java.util.HashMap;

public class Main {
    public static void main(String[] args) {
        HashMap<String, Integer> ages = new HashMap<>();
        ages.put("Ada", 28);
        ages.put("Bob", 35);
        System.out.println(ages.get("Ada"));
        System.out.println(ages.getOrDefault("Cy", -1));
    }
}
```

`getOrDefault` returns a fallback value instead of `null` when the key is
missing, avoiding an extra existence check.

### Example 3

```java
import java.util.HashSet;

public class Main {
    public static void main(String[] args) {
        HashSet<String> uniqueTags = new HashSet<>();
        String[] tags = {"java", "code", "java", "tutorial"};
        for (String tag : tags) {
            uniqueTags.add(tag);
        }
        System.out.println(uniqueTags.size());
    }
}
```

Adding a duplicate value to a `HashSet` has no effect, since sets only
keep unique items, so `uniqueTags.size()` is 3, not 4.

## Common Mistakes

```java
import java.util.HashMap;

HashMap<String, Integer> ages = new HashMap<>();
int age = ages.get("Missing");
```

This throws `NullPointerException`, because `get` returns `null` for a
missing key, and auto-unboxing `null` into a primitive `int` fails. Use
`getOrDefault` or check `containsKey` first.

## Best Practices

Program against the interface type (`List<String>` rather than
`ArrayList<String>`) when declaring variables, so the underlying
implementation can be changed later with minimal code changes. Choose
`HashMap`/`HashSet` for speed when order does not matter, and
`LinkedHashMap`/`LinkedHashSet` or `TreeMap`/`TreeSet` when insertion
order or sorted order matters.

## Related Topics

Generics, Arrays, Streams

# Generics

## Overview

Generics let classes and methods work with different types while still
catching type errors at compile time, avoiding the need for manual
casting.

## Key Concepts

A generic type parameter, conventionally written as a single capital
letter like `T`, acts as a placeholder for an actual type supplied when
the class or method is used. The Collections Framework relies heavily on
generics, such as `List<String>` versus `List<Integer>`.

## Syntax

```java
public class Box<T> {
    private T content;

    public void set(T content) {
        this.content = content;
    }

    public T get() {
        return content;
    }
}
```

## Examples

### Example 1

```java
public class Box<T> {
    private T content;

    public void set(T content) {
        this.content = content;
    }

    public T get() {
        return content;
    }
}

public class Main {
    public static void main(String[] args) {
        Box<String> box = new Box<>();
        box.set("Hello");
        System.out.println(box.get());
    }
}
```

`Box<String>` fixes `T` to `String` for this particular instance, so
`get()` returns a `String` directly, without any casting needed.

### Example 2

```java
public class Pair<A, B> {
    private A first;
    private B second;

    public Pair(A first, B second) {
        this.first = first;
        this.second = second;
    }

    public A getFirst() {
        return first;
    }

    public B getSecond() {
        return second;
    }
}
```

A class can declare multiple type parameters, letting `Pair<String,
Integer>` hold two related values of different types together.

### Example 3

```java
public class Main {
    public static <T> T firstElement(java.util.List<T> list) {
        return list.get(0);
    }

    public static void main(String[] args) {
        java.util.List<String> names = java.util.List.of("Ada", "Bob");
        System.out.println(firstElement(names));
    }
}
```

A generic method declares its own type parameter, `<T>`, before the
return type, allowing it to work with a list of any type while still
being fully type-checked.

## Common Mistakes

```java
Box rawBox = new Box();
rawBox.set("Hello");
Integer number = (Integer) rawBox.get();
```

Using a "raw type" (`Box` without `<String>`) loses the compile-time type
checking that generics are meant to provide, allowing this cast to
compile but fail at runtime with `ClassCastException`. Always specify the
type parameter.

## Best Practices

Always specify a concrete type argument (such as `Box<String>`) rather
than using raw generic types. Use bounded type parameters, such as
`<T extends Comparable<T>>`, when a generic method needs to rely on
specific behavior of the type.

## Related Topics

Collections, Type Safety, Wildcards

# Streams and Lambdas

## Overview

The Stream API, combined with lambda expressions, provides a concise,
functional style for processing collections of data, similar in spirit to
JavaScript's array methods.

## Key Concepts

A lambda expression is a compact way of writing an anonymous function,
often used to implement a single-method interface. A stream represents a
sequence of elements that can be processed with operations like `filter`,
`map`, and `collect`, without modifying the original collection.

## Syntax

```java
list.stream()
    .filter(x -> x > 10)
    .map(x -> x * 2)
    .collect(Collectors.toList());
```

## Examples

### Example 1

```java
import java.util.List;
import java.util.stream.Collectors;

public class Main {
    public static void main(String[] args) {
        List<Integer> numbers = List.of(1, 2, 3, 4, 5);
        List<Integer> evenSquares = numbers.stream()
                .filter(n -> n % 2 == 0)
                .map(n -> n * n)
                .collect(Collectors.toList());
        System.out.println(evenSquares);
    }
}
```

The stream filters out odd numbers, squares the remaining ones, and
collects the results into a new list, all without a manual loop.

### Example 2

```java
import java.util.List;

public class Main {
    public static void main(String[] args) {
        List<String> names = List.of("Ada", "Bob", "Cy");
        names.forEach(name -> System.out.println("Hello, " + name));
    }
}
```

`forEach` accepts a lambda that runs once for each element, a common
alternative to a for-each loop for simple actions.

### Example 3

```java
import java.util.List;

public class Main {
    public static void main(String[] args) {
        List<Integer> numbers = List.of(1, 2, 3, 4, 5);
        int total = numbers.stream()
                .mapToInt(Integer::intValue)
                .sum();
        System.out.println(total);
    }
}
```

`Integer::intValue` is a method reference, a shorthand for a lambda that
just calls one existing method, and `mapToInt` converts the stream to a
primitive `int` stream so `sum()` can be used directly.

## Common Mistakes

```java
List<Integer> numbers = List.of(1, 2, 3);
numbers.stream().filter(n -> n > 1);
System.out.println(numbers);
```

This does nothing useful, because streams are lazy: without a terminal
operation like `collect`, `forEach`, or `sum`, the `filter` step never
actually executes, and `numbers` remains unchanged and unprinted in any
new form.

## Best Practices

Use streams for transforming and filtering collections when it improves
readability, but fall back to a plain loop for logic involving multiple
side effects or complex branching, where streams can become harder to
read than a loop.

## Related Topics

Collections, Functional Interfaces, Lambda Expressions

# File I/O

## Overview

Java provides several ways to read from and write to files, ranging from
the newer `java.nio.file` package to the classic `java.io` classes.

## Key Concepts

`Path` and `Files` (from `java.nio.file`) provide a modern, convenient
API for most file operations. Resources like file streams should be
closed after use; try-with-resources handles this automatically, even if
an exception occurs while the resource is open.

## Syntax

```java
import java.nio.file.Files;
import java.nio.file.Path;

String content = Files.readString(Path.of("data.txt"));
```

## Parameters

- `Files.readString(path)`: reads an entire file's contents as a
  `String`.
- `Files.writeString(path, content)`: writes a `String` to a file,
  creating or overwriting it.
- try-with-resources: `try (Resource r = ...) { }` automatically closes
  `r` at the end of the block.

## Examples

### Example 1

```java
import java.nio.file.Files;
import java.nio.file.Path;
import java.io.IOException;

public class Main {
    public static void main(String[] args) {
        try {
            Files.writeString(Path.of("notes.txt"), "Hello from Java\n");
            String content = Files.readString(Path.of("notes.txt"));
            System.out.println(content);
        } catch (IOException e) {
            System.out.println("File error: " + e.getMessage());
        }
    }
}
```

`Files.writeString` and `Files.readString` handle the full round trip of
writing and reading a small text file in just two lines each.

### Example 2

```java
import java.io.BufferedReader;
import java.io.FileReader;
import java.io.IOException;

public class Main {
    public static void main(String[] args) {
        try (BufferedReader reader = new BufferedReader(new FileReader("data.txt"))) {
            String line;
            while ((line = reader.readLine()) != null) {
                System.out.println(line);
            }
        } catch (IOException e) {
            System.out.println("Could not read file: " + e.getMessage());
        }
    }
}
```

Try-with-resources ensures `reader` is closed automatically once the
block finishes, even if `readLine()` throws an exception partway through.

### Example 3

```java
import java.nio.file.Files;
import java.nio.file.Path;
import java.io.IOException;
import java.util.List;

public class Main {
    public static void main(String[] args) {
        try {
            List<String> lines = Files.readAllLines(Path.of("data.txt"));
            for (String line : lines) {
                System.out.println(line.toUpperCase());
            }
        } catch (IOException e) {
            System.out.println("File error: " + e.getMessage());
        }
    }
}
```

`Files.readAllLines` reads a whole file into a `List<String>`, one entry
per line, convenient for smaller files that comfortably fit in memory.

## Common Mistakes

```java
FileReader reader = new FileReader("data.txt");
BufferedReader buffered = new BufferedReader(reader);
String line = buffered.readLine();
```

Forgetting to close `buffered` (or better, forgetting to use
try-with-resources) can leave the file locked by the program, or lose
buffered data if the program exits unexpectedly before `close()` is
called.

## Best Practices

Prefer `java.nio.file.Files` convenience methods for straightforward,
whole-file reads and writes, and always use try-with-resources when
manually opening a stream or reader for more advanced or line-by-line
processing.

## Related Topics

Exception Handling, Streams, Paths

# Packages and Imports

## Overview

A package groups related classes together under a common namespace,
helping organize larger Java codebases and avoid naming conflicts between
classes with the same name in different libraries.

## Key Concepts

A package is declared at the top of a file with `package name;` and
usually corresponds to a directory structure on disk. Classes from other
packages are brought into scope with `import`, and classes in the special
`java.lang` package, such as `String` and `Math`, are available
automatically without any import.

## Syntax

```java
package com.example.myapp;

import java.util.List;
import java.util.ArrayList;
```

## Examples

### Example 1

```java
package com.example.shapes;

public class Circle {
    private double radius;

    public Circle(double radius) {
        this.radius = radius;
    }

    public double area() {
        return Math.PI * radius * radius;
    }
}
```

`package com.example.shapes;` places this class inside that package,
distinguishing it from any unrelated `Circle` class that might exist in a
different package.

### Example 2

```java
package com.example.app;

import com.example.shapes.Circle;

public class Main {
    public static void main(String[] args) {
        Circle circle = new Circle(5.0);
        System.out.println(circle.area());
    }
}
```

The `import` statement lets `Main` refer to `Circle` by its short name,
instead of writing the fully qualified `com.example.shapes.Circle` every
time.

### Example 3

```java
import java.util.*;

public class Main {
    public static void main(String[] args) {
        List<String> names = new ArrayList<>();
        names.add("Ada");
        Map<String, Integer> ages = new HashMap<>();
        ages.put("Ada", 28);
    }
}
```

A wildcard import (`java.util.*`) brings in every public class from that
package, though explicitly importing only what is used is generally
considered clearer.

## Common Mistakes

```java
package com.example.app;

public class Helper {
}
```

If this file is not actually placed inside a matching directory structure
(such as `com/example/app/Helper.java`), many build tools will fail to
compile or locate the class correctly, since Java expects the package
declaration to match the file's physical location.

## Best Practices

Follow the convention of reversed domain names for package names, such as
`com.company.project`, to avoid naming collisions with other
organizations' code. Prefer explicit imports over wildcard imports in
most cases, since they make it clear exactly which classes a file depends
on.

## Related Topics

Classes, Access Modifiers, Build Tools
